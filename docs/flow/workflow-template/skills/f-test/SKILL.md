# 独立测试 Skill

本 Skill 是一份流程指引——独立于开发流程的测试工具，支持三个命令。由 Claude 在主对话中直接执行，通过读取 CLAUDE.md 的 Testing 部分获取项目测试规范。

## 前置检查

读取 CLAUDE.md 的 Capabilities 节，检查 `testing` 标签值：

- `testing: false` → 提示"测试框架未配置"，建议配置后运行 `/f-init -u`，然后退出
- 非 false → 正常执行

## 命令总览

| 命令 | 用途 |
|------|------|
| `/f-test run [范围]` | 运行测试并报告结果 |
| `/f-test cover [范围]` | 分析测试覆盖率 |
| `/f-test write [目标]` | 为已有代码补充测试 |

## `/f-test run [范围]`

### 参数

| 参数 | 说明 |
|------|------|
| 无参数 | 运行全量测试 |
| `frontend` / `backend` | 按端过滤 |
| `<文件路径>` | 运行指定文件的测试 |
| `<模块名>` | 智能匹配相关测试文件 |

可追加类型过滤：`unit` / `integration` / `e2e`

### 流程

1. 读取 CLAUDE.md Testing 部分，确定测试命令和约定
2. 根据参数构造具体的测试命令
   - 文件路径 → 确认文件存在，查找对应测试文件
   - 模块名 → 用 Grep/Glob 查找匹配的测试文件，多个则让用户选择
3. 执行测试命令
4. 分析输出，展示结果表格（通过/失败/跳过/耗时）
5. 如有失败，展示失败详情（文件位置、期望 vs 实际）

## `/f-test cover [范围]`

### 参数

与 `run` 相同（无参数 / frontend / backend）

### 流程

1. 读取 CLAUDE.md Testing 部分，确定覆盖率命令
2. 执行覆盖率测试
3. 分析输出，展示覆盖率摘要表格（statements / branches / functions / lines）
4. 列出覆盖率最低的 Top 10 文件
5. 询问用户是否为低覆盖文件补充测试 → 如需要，进入 `write` 流程

## `/f-test write [目标]`

### 参数

| 参数 | 说明 |
|------|------|
| `<文件路径>` | 为指定文件编写测试 |
| `<模块名>` | 为指定模块编写测试 |
| 无参数 | 分析覆盖率，自动选择最需要测试的文件 |

### 流程

1. **确定目标**：定位源文件和现有测试文件（如有）
2. **分析代码**：
   - 导出的函数/类/组件及其签名
   - 分支逻辑、边界条件
   - 依赖关系和 Mock 需求
   - 已有测试覆盖的场景（如有测试文件）
3. **设计测试计划**：向用户展示新增用例列表和 Mock 策略 → 确认
4. **编写测试**：
   - 遵循 CLAUDE.md Testing 部分的约定（环境、import 方式、文件放置）
   - AAA 模式（Arrange-Act-Assert）
   - 描述性测试名称（`should xxx when yyy`）
   - 已有测试文件 → 追加；无 → 创建新文件
5. **验证**：运行测试确保全部通过（失败则修复，最多 3 次）
6. **输出摘要**：目标文件、测试文件、新增用例数、测试结果
