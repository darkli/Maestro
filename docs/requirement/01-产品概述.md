# 01 - 产品概述

## 1.1 项目定位

Maestro 是一个 **Python CLI 工具**，核心功能是使用可配置的 Manager Agent（外层决策 AI）自动驱动编码工具（Claude Code / Codex CLI / Gemini CLI 等）完成开发任务。

**一句话描述**：AI 管 AI —— 用便宜的 LLM 当"项目经理"，指挥昂贵的编码 AI 干活。

## 1.2 核心价值

| 价值 | 说明 |
|------|------|
| 无人值守开发 | VPS 后台运行（nohup + PID 管理），SSH 断开不中断，任务从下达到完成全自动 |
| 成本可控 | 外层 Manager 可用 DeepSeek/Ollama 等低成本模型，编码层可按任务切换 Claude / Codex |
| 远程控制 | 通过 Telegram Bot 在手机上下达任务、查看进度、发送反馈 |
| 崩溃恢复 | Checkpoint 机制，进程崩溃后可从断点恢复 |
| 多模型支持 | Manager 支持 OpenAI/Anthropic/DeepSeek/Gemini/Ollama/Azure 六种 provider |

## 1.3 目标用户

- 需要长时间运行 AI 编码任务的独立开发者
- 在 VPS 上部署 AI 开发环境的远程工作者
- 希望通过 Telegram 在手机上控制 AI 编码的用户

## 1.4 系统边界

### 系统负责的

- 管理编码任务的完整生命周期（创建 → 执行 → 完成/失败）
- 调用外层 LLM 分析编码工具输出并决定下一步
- 多任务并行管理和进度追踪
- Telegram 远程交互和状态推送
- VPS 部署和进程保活（nohup + PID 文件）
- 崩溃恢复和安全熔断

### 系统不负责的

- 不提供 Web UI（仅 CLI + Telegram）
- 不管理 LLM API 账户和计费
- 不提供代码审查功能（由编码工具自身处理）
- 不支持多用户 / 多租户
- 不提供数据库存储（纯文件 IPC）

## 1.5 核心架构概览

```
用户 ──→ CLI / Telegram ──→ Orchestrator ──→ ┌─ ManagerAgent（决策）
                                              └─ ToolRunner（执行）
                                                     ↓
                                              Claude Code / Codex CLI / Gemini CLI
```

**核心闭环**：
1. 用户通过 CLI 或 Telegram 下达需求
2. Orchestrator 启动主循环
3. ToolRunner 调用编码工具执行指令
4. ManagerAgent 分析输出，决定下一步（execute/done/blocked/ask_user/retry）
5. 循环直到任务完成或触发熔断

## 1.6 技术栈

| 层面 | 技术 |
|------|------|
| 语言 | Python 3.10+ |
| CLI 框架 | argparse（标准库） |
| LLM 调用 | openai SDK / anthropic SDK |
| 异步框架 | asyncio（Telegram Bot） |
| 进程管理 | subprocess / nohup |
| 配置 | YAML + dataclass |
| 部署 | deploy.sh（Bash 脚本） |
| 测试 | pytest |

## 1.7 关键设计约束

| 约束 | 原因 |
|------|------|
| 纯文件 IPC，不用数据库 | 简单可靠，VPS 部署零依赖 |
| subprocess 调用编码工具，不用 pexpect | 每轮独立进程，稳定性好 |
| JSON Action 协议 | Manager 输出可结构化解析，支持 Fallback |
| Prompt 外置到 Markdown 文件 | 支持热加载，修改无需重启 |
| nohup + PID 文件进程保活 | 简单可靠，不依赖 systemd（任务层） |
