# 需求分析：Focus 自动恢复

## 核心问题

任务因 ask_user 超时（或其他原因）进入 FAILED 状态后，Orchestrator 进程已退出。用户通过 Telegram 发消息时，消息写入 inbox.txt 但无人消费，形成"死信"。

## 功能需求

### FR-1: 向死任务发消息时自动恢复

**描述**: 当用户通过 `/ask` 或直接回复向一条已停止的任务发送消息时，Daemon 自动检测进程不存在，触发 resume 流程。

**触发条件**:
- 任务状态为 `FAILED` 或 `WAITING_USER`（进程已死）
- 用户发送了消息（通过 `/ask`、直接回复通知、或 focus 后发普通消息）

**排除条件**:
- 状态为 `COMPLETED`：任务已正常完成，无需恢复
- 状态为 `ABORTED`：用户主动终止，不应自动恢复（如需恢复应显式 `/resume`）
- 状态为 `EXECUTING` 且进程存活：正常运行中，消息正常写入 inbox

**行为**:
1. 将用户消息写入 inbox.txt
2. 在后台启动 resume 进程
3. 回复用户："任务已自动恢复，你的消息已送达"
4. 恢复后的 Orchestrator 读取 inbox.txt 获取用户消息

### FR-2: focus 死任务时提示可恢复

**描述**: 当用户 `/focus` 一条已停止的任务时，除了显示当前状态，额外提示"发送消息即可自动恢复"。

**行为**:
- 状态为 `FAILED`: 显示失败原因 + "直接发消息即可自动恢复任务"
- 状态为 `COMPLETED`: 正常显示，不提示恢复
- 状态为 `ABORTED`: 显示 "任务已终止，如需恢复请使用 /resume"

### FR-3: 恢复时注入用户消息到 Manager 上下文

**描述**: 自动恢复的任务在 resume 时，应将 inbox.txt 中的用户消息包含在 resume notice 中，确保 Manager 能看到用户的回复，而不是仅通知"崩溃恢复"。

**当前 resume notice**:
```
[系统通知] 任务从第 N 轮崩溃恢复。上一条指令是：XXX。请决定下一步操作。
```

**增强后 resume notice**:
```
[系统通知] 任务从第 N 轮恢复。上一条指令是：XXX。
用户回复了：<inbox 中的消息>
请基于用户的回复决定下一步操作。
```

### FR-4: 进程存活检测

**描述**: 判断任务进程是否存活，避免对正在运行的任务重复启动 resume。

**检测方式**: 读取 state.json 中的 `worker_pid`，用 `os.kill(pid, 0)` 检测。

**注意**: Daemon 的 `_monitor_loop` 中已有类似逻辑（第 611-633 行），检测到死进程会将状态设为 FAILED。

## 非功能需求

- NFR-1: 自动恢复只触发一次 — 避免用户连续发多条消息导致多次 resume
- NFR-2: 恢复后 focus 状态保持 — 用户发消息触发恢复后，该任务仍然是 focused
- NFR-3: 兼容性 — 不影响现有的 `/ask` 对运行中任务的正常行为

## 用户故事

### US-1: 隔天回复超时任务
```
Day 1:
  Bot: [abc123] 需要你的回复 - git 身份验证失败，请配置后回复
  （用户未回复，ask_user 超时，任务变为 FAILED）

Day 2:
  用户: /focus abc123
  Bot: 已关注任务 [abc123]
       需求: 修复登录 Bug
       状态: 失败（等待用户回复超时）
       直接发消息即可自动恢复任务

  用户: 已配置好 git 身份验证，请继续
  Bot: 任务 [abc123] 已自动恢复，你的消息已送达
  （后台启动 resume，Manager 看到用户回复后继续执行）
```

### US-2: 直接回复通知消息
```
Bot 推送: [abc123] 任务失败
          原因: 等待用户回复超时
          恢复: maestro resume abc123

用户直接回复该消息: 我已经配置好了，继续
Bot: 任务 [abc123] 已自动恢复，你的消息已送达
```

### US-3: 对运行中任务发消息（行为不变）
```
用户: /ask abc123 用 TypeScript 写
Bot: 已向任务 [abc123] 发送反馈: 用 TypeScript 写
（消息写入 inbox.txt，正在运行的 Orchestrator 正常消费）
```
